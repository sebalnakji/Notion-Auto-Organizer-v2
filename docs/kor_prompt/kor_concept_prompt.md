# 개념 정리 프롬프트

## 목표

현업 개발자가 개념을 정확히 정립하고 깊이 이해하기 위한 레퍼런스 노트를 작성한다.
독자는 기본적인 프로그래밍 경험이 있으나, 해당 주제에 대한 깊은 지식은 없을 수 있다고 가정한다.

---

## 구조

필수: Why → Where → What → How
선택: Pitfall, Tips, Reference (해당하는 내용이 있을 때만)

- Why: 기술적 배경과 이 개념이 해결하는 문제
- Where: 실제 사용되는 상황 2~3가지 — 각 상황에서 왜 이 개념이 필요한지 맥락 포함
- What: 정확한 기술적 정의 + 내부 동작 원리. 비유는 보조 수단으로만 사용 (핵심)
- How: 단계별 작동 방식 + 실무 시나리오 기반 실제 예시 (핵심)
- Pitfall: 실무에서 흔히 발생하는 실수나 오해 2~3가지
- Tips: 공식 문서에 잘 안 나오는 실무 노하우 — 성능/설계 팁, 관용적 패턴, 유용한 컨벤션
- Reference: 공식 문서, RFC, 주요 기술 블로그, 관련 스펙 문서

What과 How가 가장 중요하다. 이 두 섹션이 불명확하면 전체가 흔들린다.

---

## 단계 1 — Claude: 초안 작성

```
주제: {주제명}
작업: 개념 정리 초안 작성 후 src/concept_workspace/{대주제}/{중주제}/{개념명}.md 저장

구조를 순서대로 따른다.
- What, How에 가장 많은 비중을 둔다.
- What: 정확한 정의와 내부 동작 원리 중심으로 작성한다. 비유는 보조 수단으로만 사용한다.
- How는 반드시 실제 예시(코드/시나리오/다이어그램 중 현업 개발자에게 가장 적합한 것)를 포함한다.
- Pitfall, Tips, Reference는 해당하는 내용이 있을 때만 포함한다.

조건:
- 검증된 사실만 포함한다. 불확실한 내용은 제외한다.
- 현업 개발자 수준에 맞춰 작성한다. 입문자용으로 낮추지 않는다.
- Notion 토글 변환이 쉽도록 계층 구조를 명확히 한다.
- 공식 문서나 신뢰할 수 있는 링크가 있으면 포함한다.
- 작성 후 스토리 흐름과 팩트를 자체 검토하고 저장한다.
- 문서는 한글로 작성한다. 단, RAG, LLM 등 전문 용어와 고유명사는 원문 그대로 사용한다.
```

## 단계 2 — Gemini: 보완

```
파일: @{개념명}.md
작업: 스토리 흐름을 유지하며 문서 품질을 높인다.

점검 순서:
1. Why: 기술적 배경이 문제의 기원을 설명하기에 충분한가? 부족하면 깊이를 더한다.
2. Where: 시나리오가 구체적이고 해당 개념이 왜 필요한지 맥락이 명확한가? 모호하면 보강한다.
3. What [최우선]: 기술적 정의가 정확한가? 내부 동작 원리가 명확히 설명되었는가? 가치 없는 과도한 비유는 제거한다.
4. How [최우선]: 단계별 설명이 실무 수준인가? 예시가 현실적인가? 너무 기초적이면 개선한다.
5. Pitfall (있을 때만): 입문자 실수가 아닌 실무에서 실제로 발생하는 문제인가? 검증되지 않은 내용은 삭제한다.
6. Tips (있을 때만): 실제로 유용한 실무 노하우인가? 문서에서 쉽게 찾을 수 있는 뻔한 내용은 삭제한다.
7. Reference (있을 때만): 링크가 유효하고 신뢰할 수 있는 출처인가? 공식 문서가 빠져 있으면 추가한다.

조건:
- 불확실한 내용은 삭제한다.
- Why → Where → What → How 흐름이 끊기지 않도록 재배치한다.
- 중요 정보는 `> ` 또는 `!> `로 강조한다.
- 불필요한 수식어를 제거하고 간결하게 유지한다.
- 검토 후 저장한다.
```

## 단계 3 — Claude: 최종 검증 및 보완

```
파일: @{개념명}.md
작업: 최종 검증 및 부족한 부분을 보완한다.

검증:
- Why/Where: 흐름이 자연스럽게 연결되고 기술적 맥락이 충분한가?
- What [최우선]: 정의가 정확하고 내부 동작 원리가 명확히 설명되었는가?
- How [최우선]: 단계별 설명이 구체적이고 실무 수준의 예시가 있는가?
- Pitfall/Tips/Reference (있을 때만): 내용이 정확하고 유용하며 유효한가?
- 오해를 부르는 문장, 입증 불가한 주장, 유효하지 않은 링크를 제거한다.
- 오탈자와 어색한 문맥을 수정한다.
```
