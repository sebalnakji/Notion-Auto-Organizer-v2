# 논문/파일 정리 프롬프트

## 유형

- 자료조사: 핵심 주제에 대해 여러 자료를 수집·검토. 적합성을 먼저 판단한 후 정리한다.
- 연구: 논문/파일을 요약본 + 상세본으로 깊이 있게 정리한다.

---

## 자료조사

### 흐름

```
초록 → 1차 검토 의견 → 사용자 판단
  NO → 파일 삭제, 종료
  YES → 전체 읽기 → 최종 검토 의견 → 사용자 판단
          NO → 파일 삭제, 종료
          YES → 초안 작성 → 저장
```

### 단계 1 — Claude: 적합성 검토 + 초안 작성

```
핵심 주제: {핵심 주제명}
자료: @{파일명}

[A. 1차 검토 — 초록만 읽는다]
아래 형식으로 의견을 제시하고 사용자 판단을 기다린다.
응답 전에 다음 단계로 넘어가지 않는다.

> 1차 검토
> 제목: {제목}
> 초록 요약: {한 줄}
> 관련도: {높음/보통/낮음} — {이유}
> 권장: {진행/제외}
> 전체 내용을 읽고 계속 진행할까요?

- NO → 파일 삭제 후 종료
- YES → B로 진행

[B. 최종 검토 — 전체를 읽는다]
아래 형식으로 의견을 제시하고 사용자 판단을 기다린다.
응답 전에 다음 단계로 넘어가지 않는다.

> 최종 검토
> 핵심 주장: {한 줄}
> 초록과 차이: {있음/없음} — {내용}
> 최종 관련도: {높음/보통/낮음} — {이유}
> 권장: {정리/제외}
> 이 자료를 정리할까요?

- NO → 파일 삭제 후 종료
- YES → C로 진행

[C. 초안 작성]
src/research_workspace/{대주제}/{중주제}/{논문제목}.md 파일을 작성하고 저장한다.

1. 자료 기본 정보: 제목, 저자, 출처, 발행연도
2. 핵심 주제와의 연관성: {핵심 주제명}에 어떻게 기여하는지 2~3줄
3. 핵심 내용 [가장 중요]: 핵심 주장·근거·결론. 전문 용어는 쉬운 언어로 풀어서 설명한다.
4. 주요 데이터/근거: 핵심 주장을 뒷받침하는 수치·실험 결과·사례
5. 활용 포인트 [선택]: 연구/프로젝트 활용 방안

조건:
- 자료의 실제 내용만 기반으로 작성한다. 추측과 임의 해석은 제외한다.
- Notion 토글 변환이 쉽도록 계층 구조를 명확히 한다.
- 원문과 일치하는지 자체 검토 후 저장한다.
- 문서는 한글로 작성한다. 단, RAG, LLM 등 전문 용어와 고유명사는 원문 그대로 사용한다.
```

### 단계 2 — Gemini: 보완

```
파일: @{논문제목}.md
핵심 주제: {핵심 주제명}

점검:
1. 자료 기본 정보: 누락 항목이 있으면 보완한다.
2. 핵심 주제와의 연관성: 불명확하면 보완한다.
3. 핵심 내용 [최우선]: 주장·결론이 명확한가? 전문 용어가 설명 없이 쓰였는가? 원문과 다른 표현이 있는가?
4. 주요 데이터/근거: 수치와 출처가 정확한가?
5. 활용 포인트 (있을 때만): 구체적이고 실용적인가?

조건:
- 원문과 다른 내용은 수정한다. 불확실한 내용은 삭제한다.
- 핵심 수치나 주장은 `> ` 또는 `!> `로 강조한다.
- 검토 후 저장한다.
```

### 단계 3 — Claude: 최종 검증 및 보완 + Notion 업로드

```
파일: @{논문제목}.md
작업: 최종 검증 후 Notion 업로드 (업로드 규칙은 main_prompt.md 참조)
내용 확인 후 보완할 점이 있으면 보완한다.

검증:
- 자료 기본 정보가 완전한가?
- 핵심 주제와의 연관성이 명확한가?
- 핵심 내용 [최우선]: 원문과 일치하고 명확하게 전달되는가?
- 데이터/근거가 정확하게 인용됐는가?
- 원문과 다른 표현, 불확실한 내용을 제거한다.
- 오탈자와 어색한 문맥을 수정한다.
```

## 연구

### 단계 1 — Claude: 초안 작성

```
자료: @{파일명}
작업: 연구 정리 초안 작성 후 src/research_workspace/{대주제}/{중주제}/{논문제목}.md 저장

[요약본] — 3분 안에 핵심을 파악할 수 있어야 한다.
- 연구 목적: 무엇을 밝히려 했는가 (1~2줄)
- 핵심 발견: 가장 중요한 결과 (2~3줄)
- 시사점: 해당 분야에서 갖는 의미 (1~2줄)

[상세본]
1. 서론: 연구 배경과 문제 제기, 연구 필요성
2. 방법론 [가장 중요]: 연구 방법·실험 설계·데이터 수집 단계별 정리. 전문 용어는 쉬운 언어로 풀어서 설명한다.
3. 결과 [가장 중요]: 주요 실험 결과와 수치, 핵심 데이터
4. 결론: 최종 결론·한계점·후속 연구 방향
5. 추가 인사이트 [선택]: 특히 주목할 만한 독창적 관점

조건:
- 논문의 실제 내용만 기반으로 작성한다. 임의 해석과 추측은 제외한다.
- Notion 토글 변환이 쉽도록 계층 구조를 명확히 한다.
- 논문 내 인용 출처가 있으면 포함한다.
- 요약본과 상세본이 일치하는지 자체 검토 후 저장한다.
- 문서는 한글로 작성한다. 단, RAG, LLM 등 전문 용어와 고유명사는 원문 그대로 사용한다.
```

### 단계 2 — Gemini: 보완

```
파일: @{논문제목}.md

요약본 점검:
- 목적·발견·시사점이 각각 명확하게 전달되는가?
- 요약본과 상세본이 일치하는가? 불일치하면 즉시 수정한다.

상세본 점검:
1. 서론: 배경과 필요성이 명확한가?
2. 방법론 [최우선]: 단계별 설명이 명확한가? 전문 용어가 설명 없이 쓰였는가?
3. 결과 [최우선]: 수치와 데이터가 정확하게 인용됐는가? 원문과 다르게 해석됐는가?
4. 결론: 한계점과 후속 방향이 포함됐는가?
5. 추가 인사이트 (있을 때만): 원문 기반으로 정확한가?

조건:
- 원문과 다른 내용은 수정한다. 불확실한 내용은 삭제한다.
- 핵심 수치나 결론은 `> ` 또는 `!> `로 강조한다.
- 검토 후 저장한다.
```

### 단계 3 — Claude: 최종 검증 및 보완 + Notion 업로드

```
파일: @{논문제목}.md
작업: 최종 검증 후 Notion 업로드 (업로드 규칙은 main_prompt.md 참조)
내용 확인 후 보완할 점이 있으면 보완한다.

검증:

- 결과 [최우선]: 수치와 데이터가 정확하게 인용됐는가?
- 결론: 한계점과 후속 방향이 포함됐는가?
- 원문과 다른 해석, 과장된 표현, 불확실한 내용을 제거한다.
- 오탈자와 어색한 문맥을 수정한다.
```
